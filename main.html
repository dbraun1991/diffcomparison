Page Architecture

This portfolio website is built with a modern architecture combining static hosting and serverless edge computing. The following diagrams illustrate the system design, data flows, and component interactions that power the tracking and analytics features.

System Context Diagram

High-level overview showing system boundaries, external actors, and component relationships. This follows the C4 Model architecture documentation standard.

Components: Website Visitor (Browser) → GitHub Pages (Static Hosting) → Cloudflare Worker (Backend) → Cloudflare KV (Storage) → Administrator (Dashboard)

Communication Sequence Flow

Detailed step-by-step interaction flow showing how tracking pixels are sent, session management works, and data is persisted across the system.

Key Events: Login → UUID Generation → Page Load Tracking → Navigation Click Tracking → Dashboard Analytics

Architecture Highlights

  • Separation of Concerns: Static frontend (GitHub Pages) + Serverless backend (Cloudflare Workers)
  • Edge Computing: Global low-latency responses via Cloudflare's edge network
  • Privacy-First Design: Session-based UUID v4 tracking, no persistent cookies
  • Scalability: Serverless architecture with automatic scaling
  • Observability: Real-time dashboard with search and filtering capabilities
  • Standards Compliance: Architecture documented using C4 Model and UML standards
